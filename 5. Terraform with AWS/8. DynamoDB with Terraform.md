# DynamoDB with Terraform

> In this guide, you will learn how to create and manage DynamoDB tables using Terraform. 


## Creating the DynamoDB Table
-   In this example, we create a table named “cars” that holds data about vehicles. 
-   The table requires a name and a hash key that serves as the primary key—in our case, the vehicle identification number (VIN). 
- Additionally, an attribute block is specified to detail the attribute (VIN) and the billing mode is set to on-demand (PAY_PER_REQUEST).


```bash
resource "aws_dynamodb_table" "cars" {
  name         = "cars"
  hash_key     = "VIN"
  billing_mode = "PAY_PER_REQUEST"
  attribute {
    name = "VIN"
    type = "S"
  }
}
```

When using provisioned capacity (the default mode), you would need to declare the read and write capacity units. However, this example uses `PAY_PER_REQUEST` (on-demand mode).
```bash
terraform apply
```

## Inserting Items into the Table

Terraform provides the aws_dynamodb_table_item resource for adding entries to your DynamoDB table.
-    This resource requires the table name and hash key (which can be referenced from the table resource) along with the item data.

```bash
resource "aws_dynamodb_table" "cars" {
  name         = "cars"
  hash_key     = "VIN"
  billing_mode = "PAY_PER_REQUEST"
  attribute {
    name = "VIN"
    type = "S"
  }
}

resource "aws_dynamodb_table_item" "car-items" {
  table_name = aws_dynamodb_table.cars.name
  hash_key   = aws_dynamodb_table.cars.hash_key
  item       = <<EOF
{
  "Manufacturer": {"S": "Toyota"},
  "Make": {"S": "Corolla"},
  "Year": {"N": "2004"},
  "VIN": {"S": "4Y1SL65848Z411439"}
}
EOF
}
```

Apply the configuration to insert the item:

```bash
terraform apply
```
