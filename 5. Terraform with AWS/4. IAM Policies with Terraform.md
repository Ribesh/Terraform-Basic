# IAM Policies with Terraform
>In this lesson, you will learn how to create IAM policies using Terraform and attach them to an AWS user.


## Creating an IAM Policy Document
AWS uses JSON-formatted policy documents to define permissions. Below is an example of an administrator access policy document:

```bash
{
    "Version" : "2012-10-17",
    "Statement" : [
        {
            "Effect" : "Allow",
            "Action" : "*",
            "Resource": "*"
        }
    ]

}
```

## Defining Resource in Terraform

To add permission via Terraform, you will use the `aws_iam_policy` resource.



### Step1: Declate the IAM User and IAM Policy

```bash
resource "aws_iam_user" "admin-user" {
    name    =   "lucy"
    tags    =   {
        Description =   "Technical Team Leader"
    }
}


resource "aws_iam_policy" "adminUser" {
    name    = "AdminUsers"
    policy  =   ?
}
```

### Step2: Incorporate the Policy Document with Heredoc Syntax
One efficient method to include the policy document within your Terraform configuration is to use a heredoc.

```bash
resource "aws_iam_user" "admin-user" {
    name    =   "lucy"
    tags    =   {
        Description =   "Technical Team Leader"
    }
}


resource "aws_iam_policy" "adminuser" {
    name    =   "AdminUsers"
    policy  =   <<EOF
    {
        "Version":  "2012-10-17",
        "Statement" [
            {
                "Effect": "Allow",
                "Action" "*",
                "Resource": "*"
            }
        ]
    }
}
```


### Step3: Attaching the policy to the IAM User

```bash
resource "aws_iam_user_policy_attachment" "lucy-admin-access"{
    user    =   aws_iam_user.admin-user.name
    policy_arn  =  aws_iam_policy.adminuser.arn
}
```

### Complete Terraform Configuration
```bash
resource "aws_iam_user" "admin-user" {
  name = "lucy"
  tags = {
    Description = "Technical Team Leader"
  }
}

resource "aws_iam_policy" "adminUser" {
  name   = "AdminUsers"
  policy = <<EOF
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "*",
      "Resource": "*"
    }
  ]
}
EOF
}

resource "aws_iam_user_policy_attachment" "lucy-admin-access" {
  user       = aws_iam_user.admin-user.name
  policy_arn = aws_iam_policy.adminUser.arn
}
```

### Deploying the configuration
```bash
terraform plan
terraform apply
```


## Alternative Approach: Using an External JSON File


### Steps to Use an External File
1. Create a file name `admin-policy.json` in the same directory as you `main.tf`
2.  Move the JSON policy document into `admin-policy.json`


Update the IAM policy resource to read the policy document using the `file` function:

```bash
resource "aws_iam_policy" "adminUser" {
    name    =   "AdminUsers"
    policy  =   file("admin-policy.json")
}
```
