# AWS IAM with Terraform

## Creating an IAM User Resource


```bash
resource "aws_iam_user" "admin-user" {
    name    =   "Lucy"
    tags    =   {
        Description =   "Technical Team Leader"
    }
}
```


## Initializing Terraform and Running the Plan

```bash
terraform init
```

After initialization, if you run:
```bash
terraform plan
```

you might encounter two common issues:
1. Terraform may prompt for an **AWS region.** Although IAM resources are global, Terraform requires a region because most AWS resources are region-specific.
2. Terraform might not find valid **AWS credentials** to connect to your AWS account.


## Configuring the AWS Provider

```bash
provider "aws" {
    region  =   "us-west-2"
    access_key  =   "*****************"
    secret_key  =   "*****************"
}


resource "aws_iam_user" "admin-user" {
    name    =   "Lucy"
    tags    =   {
        Description =   "Technical Team Leader"
    }
}
```


## Executing the Terraform Plan and Apply

```bash
terraform plan
terraform apply
```

## Best Practices for Managing Credentials
>Hardcoding credentials in your Terraform configuration is not recommended, especially when storing files in version control.
> * Instead, use environment variables or CLI configurations to manage your credentials securely.

### AWS CLI Configuration
Configure the AWS CLI on you machine using:
```bash
aws configure
```

This create a credentials file at `~/.aws/credentials`
```bash
[default]
aws_access_key_id   = YOUR_ACCESS_KEY_ID
aws_secret_access_key   =   YOUR_SECRET_ACCESS_KEY
```

### Environment Variables
Alternatively, you can set environment variables for you AWS credentials and region:

```bash
export AWS_ACCESS_KEY_ID=YOUR_ACCESS_KEY_ID
export AWS_SECRET_ACCESS_KEY=YOUR_SECRET_ACCESS_KEY
export AWS_DEFAULT_REGION=us-west-2
```
